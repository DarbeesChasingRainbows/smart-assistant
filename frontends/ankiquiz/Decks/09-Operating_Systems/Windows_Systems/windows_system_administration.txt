// Windows System Administration & Active Directory Deck

What is the primary purpose of Active Directory in Windows enterprise environments?
To provide centralized authentication, authorization, and directory services for managing users, computers, and network resources across domains.

What is the basic command to open Active Directory management tools?
"dsa.msc" for Active Directory Users and Computers, or "Get-ADUser -Filter *" in PowerShell with Active Directory module.

What is the difference between Active Directory and local user accounts?
Active Directory provides centralized network authentication and policy enforcement, while local accounts exist only on individual computers.

What is the primary purpose of Active Directory Domains?
To organize network resources into logical administrative boundaries with centralized security policies and user management.

What is the basic command to create a new Active Directory domain?
"Install-ADDSForest -DomainName "corp.contoso.com" -InstallDns" in PowerShell, or use Server Manager > Add Roles > Active Directory Domain Services.

What is the difference between domains and workgroups in Windows networking?
Domains provide centralized authentication and management, while workgroups use peer-to-peer authentication without central control.

What is the primary purpose of Active Directory Organizational Units (OUs)?
To organize Active Directory objects for delegation of administration and application of Group Policy settings.

What is the basic command to create an Organizational Unit in Active Directory?
"New-ADOrganizationalUnit -Name "IT Department" -Path "DC=corp,DC=contoso,DC=com"" in PowerShell.

What is the difference between OUs and containers in Active Directory?
OUs support Group Policy inheritance and delegation, while containers are simple organizational objects without these features.

What is the primary purpose of Active Directory user accounts?
To provide individual network identities for authentication, authorization, and resource access across the domain.

What is the basic command to create a new user account in Active Directory?
"New-ADUser -Name "John Doe" -GivenName "John" -Surname "Doe" -SamAccountName "jdoe" -UserPrincipalName "jdoe@corp.contoso.com"" in PowerShell.

What is the difference between userPrincipalName (UPN) and sAMAccountName in Active Directory?
UPN provides email-style login format (user@domain), while sAMAccountName provides legacy NetBIOS-style login format.

What is the primary purpose of Active Directory groups?
To organize users for simplified permission assignment, email distribution, and administrative delegation.

What is the basic command to create a security group in Active Directory?
"New-ADGroup -Name "IT Staff" -GroupScope Global -GroupCategory Security" in PowerShell.

What is the difference between security groups and distribution groups in Active Directory?
Security groups are used for permission assignment and access control, while distribution groups are used for email distribution lists.

What is the primary purpose of Active Directory computer accounts?
To authenticate computers to the domain and enable centralized management of workstations and servers.

What is the basic command to join a computer to an Active Directory domain?
"Add-Computer -DomainName "corp.contoso.com" -Credential (Get-Credential)" in PowerShell, or use System Properties > Computer Name > Change.

What is the difference between domain-joined and workgroup computers?
Domain-joined computers authenticate against Active Directory and receive Group Policy, while workgroup computers operate independently.

What is the primary purpose of Group Policy in Windows environments?
To centrally configure and enforce settings for users and computers across Active Directory domains.

What is the basic command to open Group Policy Management Console?
"gpmc.msc" which provides interface for creating, linking, and managing Group Policy Objects.

What is the difference between local Group Policy and domain Group Policy?
Local Group Policy affects only the local computer, while domain Group Policy applies to multiple computers through Active Directory.

What is the primary purpose of Group Policy Objects (GPOs)?
To contain collections of policy settings that can be linked to sites, domains, or OUs for configuration management.

What is the basic command to create a new Group Policy Object?
"New-GPO -Name 'Workstation Security Settings'" in PowerShell, or right-click Group Policy Objects in GPMC > New.

What is the difference between computer configuration and user configuration in GPOs?
Computer configuration applies to computers regardless of who logs in, while user configuration applies to specific user accounts.

What is the primary purpose of Group Policy inheritance in Active Directory?
To allow policy settings to flow from parent containers to child containers with the ability to block or enforce inheritance.

What is the basic command to block Group Policy inheritance?
"Set-GPInheritance -Target 'OU=Workstations,DC=corp,DC=contoso,DC=com' -IsBlocked Yes" in PowerShell.

What is the difference between Block Inheritance and Enforced in Group Policy?
Block Inheritance prevents parent policies from applying, while Enforced ensures child policies override parent blocking.

What is the primary purpose of Group Policy security filtering?
To apply GPOs only to specific security groups, computers, or users for targeted policy deployment.

What is the basic command to set Group Policy security filtering?
In GPMC, select GPO > Scope tab > Security Filtering > Add, or use "Set-GPPermissions -Guid 'GPO-ID' -TargetName 'Domain Computers' -PermissionLevel GpoApply" in PowerShell.

What is the difference between security filtering and WMI filtering in Group Policy?
Security filtering applies policies based on group membership, while WMI filtering applies based on system properties and conditions.

What is the primary purpose of Group Policy Preferences?
To configure targeted settings with more granular control and item-level targeting than traditional Group Policy settings.

What is the basic command to configure Group Policy Preferences?
In Group Policy Editor, navigate to Computer/User Configuration > Preferences, or use PowerShell with Group Policy module.

What is the difference between Group Policy settings and preferences?
Settings are enforced policies that cannot be modified by users, while preferences are configurable settings that users can change.

What is the primary purpose of Windows Server Roles?
To provide specialized server functions like Active Directory, file services, web hosting, and application serving.

What is the basic command to install server roles via PowerShell?
"Install-WindowsFeature -Name AD-Domain-Services -IncludeManagementTools" for Active Directory Domain Services.

What is the difference between server roles and server features?
Roles provide major server functions like Active Directory or web services, while features provide supporting capabilities like .NET Framework.

What is the primary purpose of Windows Server Features?
To provide supporting components and capabilities that enhance server functionality and enable specific applications.

What is the basic command to install server features via PowerShell?
"Install-WindowsFeature -Name Telnet-Client -IncludeManagementTools" to install the Telnet client feature.

What is the difference between required and optional server features?
Required features are essential for core server operation, while optional features provide additional functionality for specific needs.

What is the primary purpose of Windows Server Manager?
To provide a unified console for managing server roles, features, resources, and remote servers across the infrastructure.

What is the basic command to open Server Manager?
"servermanager.exe" or access through Start > Administrative Tools > Server Manager.

What is the difference between Server Manager and individual management consoles?
Server Manager provides unified management of multiple servers and roles, while individual consoles focus on specific services.

What is the primary purpose of Windows PowerShell for server administration?
To automate administrative tasks, manage server configuration, and provide powerful scripting capabilities for Windows Server management.

What is the basic command to import Active Directory PowerShell module?
"Import-Module ActiveDirectory" which loads cmdlets for Active Directory administration.

What is the difference between PowerShell and traditional GUI administration?
PowerShell provides automation and scripting capabilities, while GUI administration offers visual interfaces for manual management.

What is the primary purpose of Windows Remote Server Administration Tools (RSAT)?
To enable administration of server features and roles from client computers without requiring direct server access.

What is the basic command to install RSAT via PowerShell?
"Get-WindowsCapability -Name RSAT* -Online | Add-WindowsCapability -Online" for Windows 10/11 RSAT installation.

What is the difference between RSAT and Server Manager?
RSAT provides remote administration from client machines, while Server Manager runs on servers for local and remote management.

What is the primary purpose of Windows Performance monitoring for servers?
To track system performance, identify bottlenecks, and optimize server resources for reliable service delivery.

What is the basic command to monitor server performance via PowerShell?
"Get-Counter '\Processor(_Total)\% Processor Time' -SampleInterval 1 -MaxSamples 10" for CPU monitoring.

What is the difference between Performance Monitor and Resource Monitor for servers?
Performance Monitor tracks long-term trends and data collection, while Resource Monitor shows real-time resource usage.

What is the primary purpose of Windows Event monitoring for servers?
To track system events, application errors, and security incidents for troubleshooting and audit compliance.

What is the basic command to filter Windows Events via PowerShell?
"Get-WinEvent -FilterHashtable @{LogName='System'; Level=2} -MaxEvents 50" for recent system errors.

What is the difference between Event Viewer and PowerShell event monitoring?
Event Viewer provides GUI-based event browsing, while PowerShell enables automated event analysis and filtering.

What is the primary purpose of Windows Backup for servers?
To protect critical data and system configurations through automated backups for disaster recovery and business continuity.

What is the basic command to configure Windows Server Backup via PowerShell?
"Install-WindowsFeature -Name Windows-Server-Backup" to install backup feature, then use "WBAdmin" cmdlets for backup management.

What is the difference between Windows Server Backup and third-party backup solutions?
Windows Server Backup provides basic backup capabilities, while third-party solutions offer advanced features like deduplication and cloud integration.

What is the primary purpose of Windows Update Services (WSUS) for servers?
To provide centralized management of Microsoft updates for servers and workloads in enterprise environments.

What is the basic command to install WSUS via PowerShell?
"Install-WindowsFeature -Name UpdateServices -IncludeManagementTools" to install WSUS server role.

What is the difference between WSUS and Windows Update for servers?
WSUS provides centralized update management and approval, while Windows Update connects directly to Microsoft servers.

What is the primary purpose of Windows Deployment Services (WDS)?
To provide network-based installation of Windows operating systems for automated deployment of workstations and servers.

What is the basic command to install WDS via PowerShell?
"Install-WindowsFeature -Name WDS-Deployment -IncludeManagementTools" to install Windows Deployment Services.

What is the difference between WDS and traditional installation methods?
WDS provides network-based automated installation, while traditional methods require manual installation with physical media.

What is the primary purpose of Windows Failover Clustering?
To provide high availability for critical applications and services through automatic failover between cluster nodes.

What is the basic command to install Failover Clustering via PowerShell?
"Install-WindowsFeature -Name Failover-Clustering -IncludeManagementTools" to install clustering feature.

What is the difference between failover clustering and load balancing?
Failover clustering provides high availability through redundancy, while load balancing distributes workload across multiple servers.

To distribute network traffic across multiple servers for improved availability and scalability of stateless applications.

What is the basic command to install Network Load Balancing via PowerShell?
"Install-WindowsFeature -Name NLB -IncludeManagementTools" to install Network Load Balancing feature.

What is the difference between NLB and failover clustering?
NLB distributes traffic for scalability, while failover clustering provides high availability through redundancy.

What is the difference between File Services and Storage Spaces?

What is the primary purpose of Windows Print Services for servers?

What is the difference between Print Services and direct printer sharing?
Print Services provide centralized management with driver distribution, while direct sharing connects printers to individual computers.

To host web applications, websites, and services with features like application pools, security, and management tools.

"Set-ADObject -Identity 'CN=Domain Controllers,DC=corp,DC=contoso,DC=com' -Replace @{auditPolicy='...'}" for audit policy configuration.

What is the difference between auditing and logging in Windows?

What is the difference between security baselines and custom security policies?
Security baselines provide industry-standard configurations, while custom policies address specific organizational requirements.

"Set-Service -Name 'Telnet' -StartupType Disabled" to disable unnecessary services for security hardening.

What is the difference between hardening and security monitoring?
Hardening prevents attacks through configuration, while monitoring detects and responds to security events.
